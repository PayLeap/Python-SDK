1. Transaction¶

The PayLeap Transaction API allows you to submit payment transactions to your processor over the Internet using PayLeaps payment gateway. The PayLeap Transaction API supports the following transaction types:

  • ProcessCreditCard
  • ProcessCheck
  • ProcessDebitCard
  • ProcessEBTCard
  • ProcessGiftCard
  • Card Validation Services
  • Custom Token Sample

This method will help to build the ProcessCreditCard request, to send the same request and after getting it’s response will extract it’s data and will return the response

Args:
transType (str): The transaction type being performed. e.g. Sale, Adjustment etc.
cardNum (str): The credit card number used for the transaction.
expDate (str): The expiration date of the credit card used for the transaction in MMYY format.
nameOnCard (str): The cardholder’s name as printed on the card.

amount (str): The dollar amount of the transaction in DDDDDDDDDD.CC format. This amount includes any tax or tip amounts specified in ExtData.
invNum (str): The invoice number used by the merchant to identify the transaction.
pnref (str): Not used for this transaction type.
zipCode (str): The cardholder’s billing ZIP code. Used for AVS.
street (str): The cardholder’s street address. Used for AVS.
cvNumber (str): The 3 to 4 digit card verification number. For American Express, four digits displayed on the front of the card; for other card types, usually three digits displayed on the back of the card.
extData (str): An XML string containing additional data for the transaction. See the following section for more information.
serverMode (str): Valid values are: PRODUCTION, UAT.
Returns:
TransactionXMLParser. Will populated using XML response.
Raises:
Exception, For all type of exceptions

2. Merchant¶

The PayLeap Merchant API allows you to submit payment transactions to your processor over the Internet using PayLeaps payment gateway.The PayLeap Merchant API supports the following services:

  • AddRecurringCreditCard: Adds a customer, contract, and credit card payment method in one call
  • AddRecurringCheck: Adds a customer, contract, and credit card payment method in one call
  • ProcessCreditCard: Processes credit card transactions within the recurring billing module
  • ProcessCheck: Processes check transactions within the recurring billing module
  • ManageCheckInfo: Adds, updates, and deletes check payment information
  • ManageCreditCardInfo: Adds, updates, and deletes credit card payment information
  • ManageContract: Adds, updates, and deletes contract information
  • ManageCustomer: Adds, updates, and deletes customer information

3. Report¶

The PayLeap Reporting API allows you to run reports over the Internet using PayLeaps reporting gateway. The PayLeap Reporting API supports the following services:

  • GetCardTrx: Retrieves card transaction details for a merchant
  • GetCardTrxSummary: Retrieves card transaction summary for a merchant
  • GetCheckTrx: Retrieves check transaction details for a merchant
  • GetInfo: Retrieves information from the web service
  • GetOpenBatchSummary: Retrieves payment type transaction summary of the current open batch for a merchant
